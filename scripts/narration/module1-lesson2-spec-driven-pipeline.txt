Before we talk about the migration pipeline, we need to talk about tool selection. Different migration problems need different operating models, and choosing the right tool before the workflow matters.

This course standardizes on SpecKit. SpecKit enforces one shared sequence: constitution, AS-IS spec, TO-BE spec, plan, tasks, and implementation. It gives every team and every instructor the same deliverable model and review points. But architects should also know where OpenSpec and B-MAD fit. OpenSpec is useful when you want lighter-weight brownfield change management with current-state specs and proposed deltas. B-MAD is useful when guided planning, multi-role collaboration, and broader agile workflow support add more value than a single prescriptive pipeline.

Neither replaces the required SpecKit path for this course. Students should understand the tradeoff: less ceremony can also mean less migration rigor.

Now, the SDD pipeline itself. Every greenfield step has a migration-specific purpose. The constitution defines how we migrate: standards, constraints, and quality gates. Specify captures what the legacy system does through a reverse-engineered spec. Clarify validates that extracted spec against subject matter expert knowledge and available evidence. Plan, Analyze, Tasks, and Implement carry the spec pair through execution.

In practice, the migration flow looks like this: discover legacy code, infrastructure, and dependencies. Reverse-engineer the AS-IS specification. Clarify unknowns and validate with SMEs. Re-specify the TO-BE cloud target. Plan waves, data migration, integration rewiring, and cutover. Then execute tasks and validate against the specs.

The critical concept is the spec pair. Every migration produces two specs: the AS-IS truth about the legacy system and the TO-BE contract for the cloud target. The delta between them is the migration.

Use SpecKit to teach and run the core migration method. Introduce OpenSpec or B-MAD later when the delivery context, team shape, or scope calls for a different operating model.
